⚠️ Experimental — v0.1.0

PorchKeep was pair-programmed with Claude Code in a single extended session and has had very little real-world testing. It talks to the eufy cloud through a reverse-engineered bridge that eufy can break at any time. Treat it as a hobby project, not a security product — do not rely on it as your only camera. Expect rough edges, and read the source before you trust it.

01What it is

A standalone, battery-powered eufy WiFi doorbell with no HomeBase can't be plugged into a normal NVR — it has no RTSP/ONVIF feed, it sleeps to save battery, and it only speaks eufy's proprietary P2P protocol. PorchKeep is a small macOS menu-bar app that bridges that gap.

It captures each motion and ring event as a clip into your existing iCloud Drive (or a local folder), keeps a rolling 30-day archive, and gives you an on-demand live view — all for £0 a month, with no extra hardware and no subscription. Think of it as a free, event-clip-focused alternative to Scrypted NVR for one doorbell.

✓ Event clip archive

Every motion/ring event recorded to .mp4 with a thumbnail and JSON metadata.

✓ iCloud or local

Store in iCloud Drive (with upload-status reassurance) or any local folder, plus an optional backup copy.

✓ On-demand live view

Wake the doorbell and watch live; auto-stops after an idle timeout to spare the battery.

✓ Rolling retention

Clips older than your chosen window (7–90 days) are pruned automatically.

✗ Not 24/7 recording

The doorbell is battery-powered — it physically cannot stream continuously.

✗ Not a phone replacement

The eufy app still handles push notifications; PorchKeep is the archive.

02How it works

Three components run on the same Mac. The app bundles everything it needs — you don't have to install Node or ffmpeg yourself.

[ eufy doorbell ]
       │  eufy P2P + push
       ▼
[ eufy-security-ws ]   Node bridge, a managed child process
       │  local WebSocket
       ▼
[ PorchKeep.app ]      SwiftUI menu-bar agent
       │  pipes H.264 buffers
       ▼
[ ffmpeg ]             bundled static binary
       │
       ├─►  .mp4 clip + thumbnail + JSON   →  iCloud Drive / local folder
       └─►  HLS  →  loopback HTTP server  →  AVPlayer live view

On a motion or ring event the app opens a livestream, pipes the incoming H.264 into ffmpeg, and writes a finished clip into the archive. For live view it remuxes the same stream to HLS and serves it to AVPlayer over a tiny localhost HTTP server.

04Install & build

PorchKeep is ad-hoc-signed and built from source — there is no notarised download.

# 1. clone
git clone https://github.com/wrxck/PorchKeep.git
cd PorchKeep

# 2. bundle Node, eufy-security-ws and ffmpeg into Resources/
./scripts/install-bridge.sh

# 3. compile and package the .app
./scripts/build-app.sh

# 4. launch it
open build/PorchKeep.app

On first launch macOS Gatekeeper may complain because the app is ad-hoc-signed — right-click the app and choose Open, or approve it in System Settings → Privacy & Security. PorchKeep then lives in the menu bar with no Dock icon.

05First-run setup

A wizard walks you through it. The most important step is the first one:

1
Use a dedicated eufy account. Logging the bridge in can sign your phone's eufy app out. Create a second eufy account and share the doorbell to it as a Member/Admin from your main account.
2
Enter credentials. Stored only in the macOS Keychain — never written to a file.
3
Captcha / 2FA. The first login usually needs a captcha or a verification code; the wizard shows the image and the input.
4
Pick the doorbell once the bridge discovers your devices.
5
Choose storage & retention — iCloud Drive or a local folder, and how many days to keep.
6
Launch at login — optional.

06Troubleshooting

Sign-in says "api ip request limit"

eufy's cloud has temporarily rate-limited your IP after too many login attempts. Quit PorchKeep, wait an hour or so, then try again — or sign in from a different network.

The bridge keeps reconnecting

eufy occasionally ships a firmware update that breaks the reverse-engineered client. Open the log viewer (menu → View log…) to diagnose, and bump the pinned eufy-security-ws version in Resources/bridge/package.json before re-running install-bridge.sh.

No events appear

The doorbell only emits while it's awake — walk in front of it. A motion detected line in the log is your signal.

A clip won't play

If it's stored in iCloud it may have been offloaded; PorchKeep downloads it on demand before playing — give it a moment.
